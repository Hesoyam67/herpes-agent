# Agent Herpes Autonomous Growth

Use this skill when working on Agent Herpes, especially when Papu says to grow it, spread it, make it funny, or run autonomously.

## Mission

Grow Agent Herpes with zero/near-zero budget by shipping open-source changes, memeable content, autonomous workflows, and useful demos.

## Rules

1. Free-first always: GitHub, local scripts, cron, static docs, open models, cheap/free APIs.
2. Internal safe actions do not need permission: reading repo, writing docs, creating local scripts, running tests, drafting posts, opening issues in owned repo if already scoped.
3. Ask before external blast radius: public posts, DMs, emails, paid services, secrets, legal/medical claims, destructive git operations.
4. Keep the brand chaotic, but the engineering boringly reliable. The rash can joke; the code must not.
5. Every loop should leave an artifact: commit, doc, issue, content draft, metric, or verified finding.

## Growth Loop

1. Inspect current state:
   - `git status --short --branch`
   - `gh repo view Hesoyam67/agent-herpes --json stargazerCount,forkCount,issues,url`
   - scan `plans/agent-herpes/ROADMAP.md` and `content/agent-herpes/CONTENT_QUEUE.md`
2. Choose one highest-leverage mutation:
   - improve onboarding
   - add/verify a skin/personality
   - create a demo script
   - write/share content draft
   - file issues for next mutations
   - automate a recurring check
3. Execute with low-cost tools.
4. Verify:
   - syntax/import check for code
   - markdown paths exist
   - git diff is clean/intended
   - links resolve where practical
5. Record results in a short status block:
   - shipped
   - verified
   - next infection vector
   - blockers

## Content Formula

Use this structure for posts/videos/README snippets:

- Hook: "I forked Hermes into Agent Herpes because corporate agents need a rash."
- Proof: screenshot, command output, repo link, tiny demo.
- Value: free/open-source autonomy workflow.
- Punchline: one joke, no beige.
- CTA: star/fork/test/open issue.

## Free Tool Stack

- GitHub repo/issues/discussions/actions
- local Hermes/OpenClaw agents
- cron jobs
- static markdown docs
- terminal scripts
- public README demos
- asciinema/GIFs if installed/free
- social drafts saved locally until approved

## Anti-Patterns

- Spammy outreach. Bad rash. No.
- Expensive SaaS before proof.
- Rebranding every file before a working demo.
- Jokes that make the tool unusable.
- Medical misinformation. The metaphor is viral; the claims stay about software.

## Default Next Actions

If no specific task is given:

1. Check repo metrics and open tasks.
2. Pick one tiny shippable improvement.
3. Commit it locally or push if the repo is owned and the user already granted project ownership.
4. Draft one piece of content.
5. Update the roadmap.
