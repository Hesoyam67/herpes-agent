# Herpes Agent tokenization decision log

Updated: 2026-05-20T13:57:52Z

## Decision

**Do not launch the Herpes Agent token yet.**

Ship repo + landing page + X/community funnel first. Use ClawPump token launch only after real attention exists.

This is not cowardice. This is not corporate beige. This is avoiding the ancient goblin mistake of launching a token into a room containing three bots, one wallet, and a tumbleweed with a Phantom extension.

## Why delay

ClawPump has attractive launch mechanics:

- gasless token launch path exists;
- creator/agent fee share is advertised;
- agent marketplace and automation features exist;
- launchpad narrative matches autonomous-agent chaos.

But token launches need distribution. Current bottlenecks are not smart-contract mechanics; they are:

- verified X posting identity;
- public website / project surface;
- community discovery;
- proof that the roast/report funnel gets reactions;
- clean token story that does not smell like “please pump my bag, dear strangers.”

## Launch conditions

Launch only when at least one of these is true:

1. X/community traction: 7–10 public posts/replies and 3+ external organic interactions.
2. GitHub traction: meaningful stars/forks/issues from people outside the immediate workspace.
3. Funnel traction: at least one inbound request for a custom roast/report, demo, or agent workflow.
4. ClawPump traction: visible ecosystem demand for agent-token tools and report bots.

## Recommended token format

If we launch, prefer a simple meme/community token with transparent utility:

- community coordination;
- bounty/reward queue for OSS contributions;
- request queue for ClawPump roasts/reports;
- experiment log for autonomous-agent revenue attempts.

Do **not** promise:

- profit;
- yield;
- passive income;
- trading performance;
- “AI will make number go up” wizard dust.

## Anti-rug framing

If launched, publish these commitments beside the token:

- This is an experiment, not financial advice.
- The repo stays open source.
- No anonymous promises of guaranteed upside.
- No fake utility; every shipped utility must live in the public repo.
- Treasury/burner-wallet actions are logged.
- The agent can be unhinged; wallet operations stay boring, documented, and bounded.

## Trigger phrase

When Papu says “launch the rash coin” or equivalent, first run this checklist:

- X identity verified?
- website link ready?
- token metadata drafted?
- image hosted?
- wallet/API key route verified without secret exposure?
- launch amount/cost clear?
- post-launch guard/report template ready?

If any answer is no, prepare the missing artifact first. Do not just yeet a coin because the dopamine goblin found the launch button.
